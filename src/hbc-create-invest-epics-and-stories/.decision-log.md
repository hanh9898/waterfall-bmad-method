# Decision Log

## Session 2026-05-13 — Initial Build (Headless)

### Intent
User wants a variant of `bmad-create-epics-and-stories` that produces pure INVEST + 3C's user stories for side-by-side comparison with the original `epics.md` (implementation-oriented format).

### Classification: Complex Workflow
The original skill uses step-file architecture (4 carved steps). This variant uses carved references for the story-generation stage (domain-heavy INVEST rules) but keeps prerequisites and validation inline — more outcome-driven than the original.

### Key Design Decisions

1. **Name: `create-invest-stories`** — No `bmad-` prefix (reserved for official). Clear intent in name.

2. **Output file: `invest-stories.md`** — Separate from `epics.md` so both can coexist for comparison.

3. **Customizable: Yes** — Template swap via `stories_template` scalar, plus standard persistent_facts/activation hooks.

4. **INVEST + 3C's constraints baked into story-generation reference** — These are the 7 rules the user specified:
   - AC = observable user behavior, no code/syntax/file paths
   - Stories max 1-2 days, Fibonacci points (1,2,3,5,8)
   - No enabler/foundation stories (DoD, not user stories)
   - Reframe "As a System" → user-centric or technical task
   - Parallel-friendly (reduce linear dependency)
   - AC references Architecture.md for HOW
   - Story Points in Fibonacci

5. **Reuses same input documents** — PRD, Architecture, UX Design from `{planning_artifacts}`.

6. **More outcome-driven than original** — Removed prescriptive step-by-step numbering; kept hard rules with body count (INVEST violations, dependency traps).

7. **Template includes Story Points field and Technical Tasks section** — structural differences from original template.
