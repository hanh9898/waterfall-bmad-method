# Decision Log — hbc-traceability

## Session 2026-05-26

### Classification: Simple Workflow
Living document updater — logic fits inline in SKILL.md with script assistance for deterministic extraction and reporting. Customizable via customize.toml.

### Key Decisions

1. **Incremental update, never remove** — Matrix is append-only for data. Each invoke adds or updates column values but never clears existing data. This ensures audit trail integrity and prevents accidental data loss.

2. **Two deterministic scripts** — `extract-trace-ids.py` handles regex-based ID extraction from any artifact (REQ-xxx, TC-xxx, entity names). `trace-report.py` parses the matrix markdown and computes coverage stats. LLM judgment reserved for semantic mapping (which design entity traces to which requirement).

3. **Four capabilities, not four separate skills** — Init, Update, Report, Audit are all aspects of the same living document. Single skill with capability routing keeps the matrix management cohesive.

4. **Phase detection from column fill state** — Instead of requiring explicit phase argument for Update, the skill detects which columns are empty and infers which phase's data to populate. Reduces friction while still confirming with user.

5. **Headless mode from day one** — Phase gate's O3 finding taught us to include headless mode upfront. JSON return schema enables cross-skill invocation (phase gate calls traceability after passing).

6. **story_id column optional** — Only populated if project uses story-level tracking. Not all HBC projects decompose requirements into user stories.

### Rejected Alternatives

- **Separate matrix per phase** — Would lose the "single source of truth" property. Merging 4 partial matrices is more error-prone than updating one living document.
- **Fully deterministic mapping** — REQ-to-design and REQ-to-code mapping requires semantic understanding that regex can't provide. LLM judgment is the right tool for this.
- **Database storage** — Markdown matrix is human-readable, git-diffable, and works in any project without infrastructure. Database adds complexity for no gain at this scale.
