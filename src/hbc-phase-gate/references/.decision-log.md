## Session 2026-05-26

### Classification: Simple Workflow
Engine pattern — logic fits inline in SKILL.md. No carved-out references needed. Customizable via customize.toml.

### Key Decisions

1. **Default checklists bundled in assets/** — Plan says checklists live in phase skills' assets/, but those skills aren't built yet. Shipping defaults here so the gate works standalone. Phase skills override via `{project-root}/_bmad/custom/hbc-phase-gate.toml` later. Aligns with 3-layer merge pattern.

2. **No Decision-Log Workspace pattern** — Gate reports are regenerated on each evaluation, not iteratively revised. No resume state needed.

3. **No headless mode** — Gate is invoked interactively from agent menus [PG]. Quality checks involve judgment that benefits from user context. Can be added later if needed.

4. **Checklist item types** — Four evaluation types cover the spectrum: FILE (deterministic existence), CONTENT (regex pattern match), METRIC (numeric threshold), QUALITY (LLM judgment). This maps to the plan's "deterministic where possible, LLM judgment where needed."

5. **gate_mode from config, not per-invocation** — Strict/lenient is a project-level decision, not a per-gate choice. Read from `{project-root}/_bmad/config.yaml` hbc section.

6. **Phase inference from agent context** — When invoked from [PG] menu within an agent, the engine infers which phase to evaluate (BA→1, Architect/QA→2, Dev→3, Tester→4). Reduces friction vs always asking.
