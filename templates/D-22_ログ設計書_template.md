## 2. ログレベル定義
(アプリケーションで利用するログレベルとその使い分けを定義します)

| レベル | 用途 |
|---|---|
| `FATAL` / `CRITICAL` | システムの継続が不可能な致命的なエラー |
| `ERROR` | 処理の継続は可能だが、明確なエラーが発生した場合 |
| `WARN` | 即時対応は不要だが、注意喚起が必要な事象 |
| `INFO` | システムの正常な動作を示す情報（主要な処理の開始/終了など） |
| `DEBUG` | 開発時にのみ使用する詳細なデバッグ情報 |
| `TRACE` | 最も詳細な情報。メソッドの入出力など |

## 3. ログフォーマット
- **共通フォーマット**: (例: JSON)
- **共通出力項目**:
  | 項目名 | 型 | 説明 |
  |---|---|---|
  | timestamp | String | ログ出力日時 (ISO 8601形式) |
  | level | String | ログレベル |
  | message | String | ログメッセージ本体 |
  | logger_name | String | ログ出力元のモジュール名など |
  | thread_name | String | スレッド名 |
  | request_id | String | リクエストを一意に識別するID |
